# Framework Selection Reference

> Quick reference for choosing the right questioning approach based on situation. The agent already knows these frameworks — this file helps it choose between them.

## Decision Tree

```
What kind of conversation is this?
│
├─ DISCOVERY (learning about something new)
│  ├─ First interaction, no context → FUNNEL (broad → narrow)
│  ├─ User has a goal but it's vague → GROW (Goal → Reality → Options → Will)
│  └─ User describes a problem → SPIN (Situation → Problem → Implication → Need-Payoff)
│
├─ ANALYSIS (understanding something deeper)
│  ├─ Answer has gaps or contradictions → SOCRATIC PROBE (clarify → assumptions → evidence)
│  ├─ Need to find root cause → 5 WHYS (iterative "what caused that?")
│  └─ Need to understand consequences → IMPLICATION CHAIN (if X then Y then Z)
│
├─ EXPLORATION (opening up possibilities)
│  ├─ User is stuck or blocked → QUESTION BURST (generate 10 questions, pick 3-4)
│  ├─ Need creative options → WHAT IF questions + perspective shifts
│  └─ Strategic planning → WHY → WHAT IF → HOW progression
│
├─ COMMITMENT (closing and confirming)
│  ├─ Checking readiness → SCALING ("1-10 how committed?")
│  ├─ Surfacing blockers → "What would raise it to a 10?"
│  └─ Confirming understanding → PARAPHRASE + CHECK
│
└─ NONE (context is sufficient)
   └─ Just work. No questions needed.
```

## Framework Quick Reference

### SPIN (Discovery)
**Use when:** User has a problem to solve and you need to understand it fully before proposing solutions.
```
Situation → "What's currently happening?"
Problem → "What's not working?"
Implication → "What does that cost you?"
Need-Payoff → "What would solving this mean for you?"
```
**Key rule:** Never jump to Need-Payoff. The Implication stage is where urgency emerges.

### GROW (Goal-Setting)
**Use when:** User has a goal but no clear path. Planning, roadmapping, project kickoffs.
```
Goal → "What do you want to achieve?"
Reality → "Where are you now? What's been tried?"
Options → "What could you do? What else?"
Will → "What will you do? When? How committed (1-10)?"
```
**Key rule:** The Will stage must produce a specific next action, not a vague intention.

### Socratic Probe (Analysis)
**Use when:** Something doesn't add up. The answer was too vague, contradictory, or assumption-heavy.
```
Clarify → "What do you mean by ___?"
Assumptions → "What are we taking for granted here?"
Evidence → "How do we know that's true?"
Perspectives → "How would someone else see this?"
Implications → "If that's true, what follows?"
```
**Key rule:** Don't run all 5 types in sequence. Pick the 1-2 that address the specific gap.

### Why → What If → How (Strategic)
**Use when:** Planning, strategy, creative direction. Needs to move from understanding to imagination to execution.
```
Why → "Why does this matter? Why now? Why this approach?"
What If → "What if we had no constraints? What if we combined X and Y?"
How → "How would we actually do this? What's step 1?"
```
**Key rule:** Spend the most time on "What If" — it's where breakthrough ideas emerge.

### 5 Whys (Root Cause)
**Use when:** Debugging, troubleshooting, understanding failures.
```
Why 1 → Surface answer (symptom)
Why 2 → Proximate cause
Why 3 → Contributing factor
Why 4 → Systemic issue
Why 5 → Root cause
```
**Key rule:** Use "What caused that?" or "What led to that?" instead of bare "Why?" to avoid sounding accusatory.

### Question Burst (Unsticking)
**Use when:** User is stuck. Brainstorming hasn't worked. Need to reframe.
```
1. Frame the challenge in 1-2 sentences
2. Generate 8-12 questions about it (no answers, no discussion)
3. Present them: "Which of these feel most important?"
4. Explore the top 2-3 the user selects
```
**Key rule:** Don't answer the questions yourself. Let the user's reaction guide where to go next.

### Scaling (Commitment)
**Use when:** Checking how serious someone is about a plan or decision.
```
"On a 1-to-10, how [committed/confident/ready] are you?"
If < 8 → "What would raise it to a 10?"
If ≥ 8 → Proceed.
```
**Key rule:** The "what would raise it?" follow-up is where the real insight lives. It surfaces objections people wouldn't volunteer.

## When NOT to Use Any Framework

- User gave a clear, specific request with enough context → just do it
- User said "just do it" or "whatever you think" → they want output, not more questions
- You're past question 7-8 → produce something and iterate
- The task is simple/routine → no questioning needed
